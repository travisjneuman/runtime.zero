runtime.zero summary

        

          runtime.zero is a pre-alpha, Rust-first terminal-native foundation for safe system management. Current public behavior is read-only or dry-run-first. The animated TUI on this page is a design reference for future local/manual review flows, not a production TUI claim.
        

        
    
          
  • Current foundation: CLI core, diagnostics, module registry contracts, manifest validation, store planning, and dry-run planning.
    • 
          
  • Safety posture: report first, dry-run first, quarantine before delete, explicit confirmation before mutation.
    • 
          
  • Module posture: substantial capabilities are explicit first-party module choices; third-party trust requires future hardening.
    • 
          
  • Unavailable today: verified downloads, direct bootstrap commands, package publishing, destructive cleanup, and a real TUI implementation.
